1. This skill only revolves around coding tasks, so drop everything to do with GIT, branching, workflows or PRs.
2. Focus on coding styles and give yourself enough references so you can consistently generate well-formated code while holding the balance with to many tokens.
3. Ensure to follow the PSR-12 coding guidelines which can be found here: https://www.php-fig.org/psr/psr-12/
4. The skill should make it clear from the beginning, that the current code base MAY be not following the guidelines, but that the goal is to move towards it and that you should do your best to follow it in new code and refactor old code when possible.

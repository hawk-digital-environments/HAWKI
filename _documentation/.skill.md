## For writing skills for: 6-Contributing.md

This document contains information both on frontend development (using the Svelte framework) and backend development (using the Laravel framework). To avoid too much tokens for either setting SPLIT the documentation int TWO skills.

**Frontend (Svelte) related**: Look into `./static/attachments/skills/hawki-frontend/`
**Backend (Laravel) related**: Look into `./static/attachments/skills/hawki-backend/`

Update/modify both locations with the relevant information and guidelines for contributing to the project.

For BOTH skills it is important to note:

1. The skills only revolves around coding tasks, so drop everything to do with GIT, branching, workflows or PRs.
2. Focus on coding styles and give yourself enough references so you can consistently generate well-formated code while holding the balance with to many tokens.
3. The skill should make it clear from the beginning, that the current code base MAY be not following the guidelines, but that the goal is to move towards it and that you should do your best to follow it in new code and refactor old code when possible.
